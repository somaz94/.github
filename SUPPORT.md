# Support

Thanks for using one of [somaz94](https://github.com/somaz94)'s open-source
projects. These guidelines apply to any repository that does not provide its
own `SUPPORT.md`.

<br/>

## Questions and help

- **Search existing issues** first — your question may already be answered.
- **Open an issue** in the relevant repository using the template that fits:
  - **Bug report** — something is not working as documented.
  - **Feature request** — a new capability or an improvement.
- Give as much context as you can: version, environment, steps to reproduce,
  and expected vs. actual behavior.

<br/>

## Security issues

Please do **not** open a public issue for security vulnerabilities. Use the
private reporting path described in [SECURITY.md](SECURITY.md).

<br/>

## Before you file

- Search open **and** closed issues to avoid duplicates.
- Read the repository's `README.md` and `CONTRIBUTING.md`.
- Keep one topic per issue so the discussion stays focused.
